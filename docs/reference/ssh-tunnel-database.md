# SSH 隧道连接服务器数据库

部署后，PostgreSQL 和 Redis 只监听 `127.0.0.1`，外部无法直连。如果想用本地数据库工具（如 Navicat）查看服务器上的数据，可以通过 SSH 隧道连接——不需要开放数据库端口，安全性更高。

::: tip 🎯 本页解决什么
读完这一页，你能用 Navicat 或其他数据库工具通过 SSH 隧道安全地连接服务器数据库。
:::

## 原理

SSH 隧道的工作方式：

1. 数据库工具先通过 SSH 连接到服务器
2. 在服务器内部建立一条到 `127.0.0.1:5432` 的转发通道
3. 本地的数据库请求通过这条通道到达服务器上的 PostgreSQL

数据库端口始终不对外暴露，所有流量都走 SSH 加密通道。

## Navicat 配置步骤

在 Navicat 中新建 PostgreSQL 连接，分两个 tab 填写。

### 常规 tab

| 字段 | 值 |
| --- | --- |
| 主机 | `127.0.0.1` |
| 端口 | `5432` |
| 初始数据库 | `ez_admin` |
| 用户名 | `ez_admin` |
| 密码 | `.env` 中 `EZ_DATABASE_PASSWORD` 的值（默认 `ez_admin_123456`） |

### SSH tab

勾选「使用 SSH 隧道」，然后填写：

| 字段 | 值 |
| --- | --- |
| 主机 | 服务器公网 IP |
| 端口 | `22` |
| 用户名 | `ubuntu` |
| 验证方式 | 密码（填服务器登录密码），或选择密钥文件 |

填写完成后点击「测试连接」，显示连接成功即可。

::: warning ⚠️ 主机不要填服务器公网 IP
「常规」tab 里的主机填 `127.0.0.1`，不是服务器 IP。因为 SSH 隧道建立后，数据库工具是在服务器本地去连数据库的。
:::

## 其他工具通用配置

如果你使用的是 DBeaver、DataGrip 等工具，原理相同，找到 SSH 隧道相关配置即可：

| 配置项 | 值 |
| --- | --- |
| 数据库 host | `127.0.0.1` |
| 数据库 port | `5432` |
| SSH host | 服务器公网 IP |
| SSH port | `22` |
| SSH user | `ubuntu` |

## Redis 同理

如果需要连接 Redis，把端口改为 `6379`，其余 SSH 配置不变。
