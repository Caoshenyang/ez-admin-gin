# 第 8 章：模块化接入规范

::: tip 🎯 这一章会做成什么
这一章会把最终版后台底座的模块接入方式讲透：不是只新增几个文件，而是把实体、DTO、Repository、Service、Handler、Routes、Policy、DataScope 一次串起来。
:::

## 本章聚焦什么

走到这一章时，前面的平台基础设施、认证链路、接口权限、数据权限、系统模块和前端管理台都已经具备。接下来要解决的是：新业务模块应该如何按最终结构落地，而不是继续把代码散落在全局目录里。

## 预计小节

- 模块目录与职责清单
- 后端模块接入流程
- 前端页面接入流程
- Policy 与菜单按钮元信息
- DataScope 接入规则
- 一个真实业务模块示例
