# 第 5 章：组织体系与数据权限

前面已经补齐了登录和接口权限。接下来要补的是企业后台真正的分水岭：组织体系和数据权限。

::: tip 本章怎么验证
本章完成后，系统应该具备部门、岗位、角色数据范围和当前登录人上下文这些基础能力，并能为后续列表查询提供稳定的数据过滤入口。
:::

## 本章会解决什么

- 为什么企业后台一定要先有部门和岗位
- 角色数据范围为什么不能和接口权限混在一起
- 当前登录人上下文里至少要放哪些信息
- 数据权限为什么更适合通过 `gorm.Scopes(...)` 落在查询链路里
- 多角色并集、自定义部门和超级管理员绕过如何统一定义

## 本章预计小节

- 组织模型设计
- 部门与岗位基础表
- 用户组织归属
- 角色数据范围模型
- Actor 上下文
- GORM Scope 数据过滤
- 超级管理员与多角色并集规则
