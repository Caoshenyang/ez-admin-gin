# 第 4 章：接口权限体系

前一章已经把登录、Token 和登录态打通了。现在要继续补的是“登录之后能访问什么”。

::: tip 本章怎么验证
本章完成后，系统应该能基于角色控制接口访问，同时让菜单和按钮权限与后端授权结果保持一致。
:::

## 本章会解决什么

- 角色为什么要有稳定的 `code`
- Casbin 如何做接口级权限放行
- 菜单、按钮和接口权限为什么要一起设计
- 前端按钮显隐和后端接口拦截如何保持一致
