# 第 4 章：通用系统模块

上一章已经完成登录、认证、角色权限和菜单权限。现在开始补齐后台底座最常用的一组系统模块：用户、角色、菜单、配置、文件和日志。

::: tip 本章怎么验证
本章完成后，后端应该能管理用户、角色、菜单和系统配置，并能记录登录日志、操作日志，具备一个后台系统投入真实项目时常见的基础管理能力。
:::

## 开始前确认

进入本章前，建议先确认下面几件事：

- 后端服务可以正常启动。
- `admin` 账号可以登录并访问受保护接口。
- `super_admin` 已经拥有默认接口权限和菜单权限。
- 数据库中的 `sys_user`、`sys_role`、`sys_user_role`、`sys_menu`、`sys_role_menu`、`casbin_rule` 已经准备好。

::: warning ⚠️ 系统模块不要只看接口能不能调通
这一章会逐步进入“管理数据”的场景。验证时除了看接口返回成功，还要确认数据库中的数据是否正确变化，例如用户状态是否更新、角色关系是否写入、菜单树是否符合预期。
:::

::: details 为什么这一章先做后端系统模块
前端管理台最终需要依赖这些接口来展示页面。先把后端能力补齐，后面做前端页面时就能直接对接真实接口，而不是只做静态页面。
:::
