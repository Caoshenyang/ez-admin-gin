# 执行计划与状态

这页只做一件事：把 `EZ Admin Gin` 当前这轮“企业级底座 + 完整版教程”重构的执行状态公开出来，方便跟踪主线是否走偏。

::: tip 🎯 怎么读这页
如果你想判断项目当前推进到哪一步，先看下面的阶段状态；如果你想跟着教程学习，还是以 [从零搭建教程](/tutorial/) 为主线。
:::

## 当前阶段判断

- 当前目标不是继续给 `v1` 打补丁，而是直接收敛到企业级完整版主线
- 当前主线要求代码和教程同步推进，不单独维护一套“简化版教程”
- 当前优先级是先稳住工程结构、组织体系和数据权限，再把系统模块和教程正文全部收口

## 阶段清单

### Phase 1：最终结构定稿

- `[已完成]` 锁定单体 + `bootstrap / platform / module` 的最终骨架方向
- `[已完成]` 新增 `cmd/server` 入口，并让运行入口收敛到 `bootstrap`
- `[已完成]` 补 `platform` 命名空间，为认证、权限、配置、数据库、日志、迁移、Redis、数据权限留出稳定落点
- `[已完成]` 认证模块入口进入最终结构，`login / me / menus / dashboard` 已收敛到 `module/auth`
- `[进行中]` 继续把旧模块按最终结构往 `module/*` 收口

验收目标：

- 仓库主入口已经是最终结构，不再继续扩大旧的扁平目录
- 后续新增能力可以明确知道该落在哪一层

### Phase 2：组织体系与数据权限

- `[已完成]` 补部门、岗位、用户岗位、角色自定义部门范围这些基础模型
- `[已完成]` 给用户和角色模型补 `department_id`、`data_scope`
- `[已完成]` 补 PostgreSQL / MySQL 的组织体系与数据权限迁移脚本
- `[已完成]` 补 `Actor` 请求上下文装载能力
- `[已完成]` `/auth/me` 返回组织与数据范围摘要
- `[已完成]` 用户列表、编辑、状态修改、角色绑定开始接入数据权限查询作用域
- `[待开始]` 抽出更稳定的 `ScopeResolver` 和资源级 `datascope.go`
- `[进行中]` 为部门、岗位和真实业务模块接入同样的数据权限链路

验收目标：

- 接口权限和数据权限已经正式拆成两条链路
- 当前登录人的部门、角色、数据范围可以被稳定读取和复用
- 至少一个核心资源已经真正按数据权限过滤

### Phase 3：核心系统模块最终化

- `[已完成]` 用户模块首个最终版示范：`entity / dto / repository / service / handler / routes / datascope`

- `[已完成]` 角色模块进入最终结构，并接入 `data_scope` 与自定义部门范围配置
- `[已完成]` 菜单模块进入最终结构，并补齐统一 `policy` 元信息落点
- `[已完成]` 系统配置模块进入最终结构，并把配置读取与 Redis 缓存同步收口到独立服务
- `[已完成]` 文件模块进入最终结构，并把上传校验、落盘和元数据写入收口到独立服务
- `[已完成]` 操作日志模块进入最终结构，并明确“中间件写入 + 模块查询”的审计边界
- `[已完成]` 登录日志模块进入最终结构，并明确“登录入口写入 + 模块查询”的审计边界
- `[已完成]` 公告模块进入最终结构，并把轻量内容管理能力纳入统一系统模块边界
- `[已完成]` 部门模块进入最终结构，并接入组织树与部门范围过滤
- `[已完成]` 岗位模块进入最终结构，组织闭环补齐到“部门 + 岗位 + 用户归属”
