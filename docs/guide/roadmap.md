# 路线图

`EZ Admin Gin` 的目标不是一次性 Demo，而是一套能长期复用的后台管理系统底座。

> 当前稳定版本：**v1.1.0** | 最后验证日期：2026-04-30

## 阶段规划

| 阶段 | 目标 | 验收标准 | 状态 |
| --- | --- | --- | --- |
| 项目初始化 | 建立可运行的仓库骨架 | `server`、`admin`、`docs`、`deploy`、`scripts` 职责清晰 | ✅ 已完成 |
| 后端基础设施 | 后端具备真实项目基础能力 | 配置、日志、数据库、Redis、响应格式和健康检查可运行 | ✅ 已完成 |
| 认证与权限 | 完成后台系统核心安全能力 | 登录、JWT、角色、权限、Casbin 控制链路可验证 | ✅ 已完成 |
| 通用系统模块 | 补齐后台底座常用能力 | 用户、角色、菜单、配置、文件、日志可管理 | ✅ 已完成 |
| 前端管理台 | 让系统可视化可操作 | 登录页、布局、动态菜单和基础管理页面可使用 | ✅ 已完成 |
| 业务模块规范 | 定义以后小项目如何接入 | 至少有一个示例模块能证明结构可复用 | ✅ 已完成 |
| 部署与复用 | 支持快速上线和复制 | Docker Compose、Nginx、初始化数据和部署文档齐备 | ✅ 已完成 |

## 当前下一步

v1.1.0 已发布，后续 v1.x 规划：

- 完善文档体验（补充使用指南、优化教程可读性）
- 新增业务模块示例（更贴近真实场景的接入案例）
- 单元测试与集成测试覆盖
- 前端国际化支持
- 更丰富的 Dashboard 数据展示
