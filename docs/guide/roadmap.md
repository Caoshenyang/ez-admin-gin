# 路线图

`EZ Admin Gin` 的目标不是一次性 Demo，而是一套面向 Java 转 Go 工程师、能长期复用并持续扩展的后台管理系统底座。

> 当前稳定版本：**v1.1.0** | 最后验证日期：2026-04-30

## 阶段规划

| 阶段 | 目标 | 验收标准 | 状态 |
| --- | --- | --- | --- |
| 项目初始化 | 建立可运行的仓库骨架 | `server`、`admin`、`docs`、`deploy`、`scripts` 职责清晰 | ✅ 已完成 |
| 后端基础设施 | 后端具备真实项目基础能力 | 配置、日志、数据库、Redis、响应格式和健康检查可运行 | ✅ 已完成 |
| 认证与权限 | 完成后台系统核心安全能力 | 登录、JWT、角色、权限、Casbin 控制链路可验证 | ✅ 已完成 |
| 通用系统模块 | 补齐后台底座常用能力 | 用户、角色、菜单、配置、文件、日志可管理 | ✅ 已完成 |
| 前端管理台 | 让系统可视化可操作 | 登录页、布局、动态菜单和基础管理页面可使用 | ✅ 已完成 |
| 业务模块规范 | 定义以后小项目如何接入 | 至少有一个示例模块能证明结构可复用 | ✅ 已完成 |
| 部署与复用 | 支持快速上线和复制 | Docker Compose、Nginx、初始化数据和部署文档齐备 | ✅ 已完成 |

## 当前下一步

`v1.1.0` 已完成七章主线和可用后台能力，当前正式进入 `v2` 升级阶段。

### v2 第一阶段：结构与模型打底

- 引入 `bootstrap / platform / module` 后端骨架
- 补部门、岗位、角色数据范围、自定义部门范围相关模型与迁移
- 同步补架构说明和数据权限参考文档

### v2 第二阶段：企业级结构落地

- 把核心模块逐步迁到 `service / repository` 分层
- 引入 `Actor + ScopeResolver + gorm.Scopes(...)` 数据权限链路
- 为系统模块和真实业务模块建立统一接入规范

### v2 第三阶段：企业级教程主线升级

- 把当前 7 章主线升级为企业级完整版 `0-1` 教程
- 明确面向 Java 转 Go 工程师的章节节奏和 Go vs Java 对照说明
- 同步完善 DDL、权限码、数据权限和升级手册
