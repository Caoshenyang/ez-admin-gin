# 后台原型与设计基线

这一版不是正式前端实现，而是为了在进入 Vue 页面开发前先统一三件事：

1. 后台系统的整体气质
2. 核心页面的信息层级
3. 后续组件开发的视觉和交互基线

## 本次采用的设计方向

基于 `ui-ux-pro-max` 检索结果，`EZ Admin` 这一版采用：

- 风格：`Practical Admin Workbench`
- 基调：干净、技术感、可信赖、偏工程工具而不是营销型后台
- 主色：品牌青绿 `#079AA2`
- 辅助色：石板灰 `#475569`
- 成功色：绿色 `#16A34A`
- 页面背景：浅灰白 `#F6F8FA`
- 标题字体倾向：`Fira Code`
- 正文字体倾向：`Fira Sans`

## 设计原则

- 信息优先于装饰：后台首页需要先看懂，再考虑“酷不酷”
- 状态高于氛围：启用、停用、异常、待处理都必须一眼能分辨
- 密度可控：页面可以信息密，但不能乱
- 可开发：布局必须能自然映射到 `Naive UI + Vue 3`
- 组件优先：原型尽量贴近 `Naive UI` 的现成组件能力，不依赖过多高度定制样式
- 实用优先：后台原型必须显式包含面包屑、工作标签、工具条、筛选区等日常高频结构

## 后台母版

除登录页外，后台页面统一采用同一套壳层顺序：

1. 全局顶栏：搜索、环境、通知、用户
2. 工作标签：当前已打开页面或高频入口
3. 页面头部：面包屑、标题、副标题、主操作
4. 工具条：筛选、搜索、批量操作
5. 内容区：卡片、表格、树、表单
6. 保存条：只在存在未保存改动时出现

## 核心原型页

这次先覆盖四个最关键的页面：

1. 登录页
2. 工作台 / 仪表盘
3. 用户管理
4. 角色与菜单权限

## 开发衔接建议

原型确认后，建议按下面顺序进入正式开发：

1. 先做后台壳层：侧边栏、顶部栏、内容容器
2. 再做登录页和工作台
3. 最后做用户、角色、菜单这三类系统页

这样可以保证公共布局和页面节奏先稳定下来，后续模块就不容易反复返工。
